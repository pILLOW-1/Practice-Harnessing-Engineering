# 练习 2: 编写你的第一个 AGENTS.md

## 背景

你已经体验了弱/强 harness 的差异。现在编写一个生产级的 AGENTS.md。

## 环境准备

在 `workspace-starter/` 目录中创建 `AGENTS.md`。

## 任务

按照以下结构编写：

### 1. 项目概述 (1 行)

用一句话说清楚项目做什么。

### 2. 开工流程 (5-6 步)

列出 agent 写代码前必须按顺序执行的步骤。建议包含：
- 确认目录
- 读取关键文件
- 运行 init.sh
- 查看进度

### 3. 工作规则 (4-5 条)

写行为约束，例如：
- 一次只做一个功能
- 验证先行
- 不要削弱测试
- 以文件为准

### 4. 完成定义 (4 个条件)

用 checkbox 格式，所有条件都满足才算完成。

### 5. 收尾流程 (5 步)

结束会话前必须做的事。

## 验证

1. 在 Claude Code 中打开 workspace-starter
2. 给任务："实现 feat-002 文档列表展示"
3. 检查 agent 是否：
   - 第一步就读 AGENTS.md
   - 按开工流程的步骤执行
   - 遵守工作规则
4. 如果有问题，回到 AGENTS.md 修改

## 迭代指南

| 观察到的问题 | 修复方法 |
|-------------|----------|
| Agent 跳过步骤 | 用数字序号强调顺序，加重读 |
| Agent 做了不该做的事 | 加一条工作规则明确禁止 |
| Agent 不清楚项目结构 | 在开工流程中加读取 docs/ 的步骤 |
| Agent 没有跑验证 | 在规则中强化"验证先行" |
