# 第 2 课: 指令子系统

## 目标

学会编写 AGENTS.md / CLAUDE.md，掌握"渐进式展开"的指令设计模式。

## 概念引入 (5 分钟)

### 问题: 为什么一个巨大的指令文件会失败

很多人写 AGENTS.md 时，试图把所有知识都塞进去。结果：

- 文件太长，agent 读不完或读到后面忘了前面
- 关键规则埋在第 800 行的一个段落里，被忽略
- 每次改一个小规则都要重新审视整个文件

**OpenAI 的经验**：`AGENTS.md` 应该是目录页，不是百科全书。100 行左右就够了，放不下就拆分到 `docs/` 目录里，让 agent 按需去读。

### 渐进式展开设计模式

```
好设计:                          坏设计:
========                         ========

AGENTS.md (~100 行)              AGENTS.md (800+ 行)
├── 开工流程 (6 步)              ├── 开工流程 (6 步)
├── 工作规则 (5 条)              ├── 工作规则 (5 条)
├── 完成定义 (4 个条件)          ├── 完成定义 (4 个条件)
└── 收尾流程 (5 步)              ├── 架构决策 (20 条)
                                    ├── 产品需求 (30 条)
                                    ├── 编码规范 (50 条)
                                    ├── API 文档 (40 条)
                                    ├── 部署流程 (30 条)
                                    └── 故障排查 (60 条)

详细文档放在 docs/ 目录:         agent 每次会话都在读
  docs/ARCHITECTURE.md            一个永远不会完整读完的
  docs/PRODUCT.md                 巨型文件
  docs/DESIGN.md
```

### AGENTS.md 核心结构

```markdown
# 项目概述（一句话说清楚这是什么）

## 开工流程    ← agent 每次开工前按顺序执行的步骤
## 工作规则    ← 行为约束，不可违反
## 完成定义    ← 所有条件都满足才算完成的硬性标准
## 收尾流程    ← 结束会话前必须做的事
```

## 实践环节 (20 分钟)

### 练习 1: 编写你的第一版 AGENTS.md

在 `workspace-starter/` 中创建 `AGENTS.md`：

1. 项目概述: 用一句话说清楚这个项目做什么
2. 开工流程: 列出 agent 写代码前必须做的 5-6 步
3. 工作规则: 写 4-5 条行为约束
4. 完成定义: 写 4 个完成条件（用 checkbox 格式）
5. 收尾流程: 写 5 步结束前流程

### 练习 2: 验证你的 AGENTS.md

1. 在 Claude Code 中打开 workspace-starter
2. 给一个任务："实现文档列表展示功能"
3. 观察 agent 是否正确读取并遵循 AGENTS.md
4. 检查：
   - agent 是否先读了 AGENTS.md？
   - 开工流程的步骤是否被执行？
   - agent 是否在声称完成前跑了验证？

### 练习 3: 迭代改进

根据练习 2 的观察，修改 AGENTS.md：

- 如果 agent 跳过了某个步骤 → 那个步骤的描述不够清晰
- 如果 agent 做了不该做的事 → 加一条工作规则
- 如果 agent 不清楚项目结构 → 在开工流程中加"读取 docs/ARCHITECTURE.md"
