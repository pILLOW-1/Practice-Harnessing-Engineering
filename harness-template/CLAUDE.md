# CLAUDE.md

你正在一个为长时实现工作设计的仓库中工作。优先保证可靠完成、跨会话连续性和显式验证，而不是表面上的速度。

## 固定工作循环

每轮会话开始时：

1. 运行 `pwd`，确认当前在仓库根目录。
2. 读取本文件 `CLAUDE.md`。
3. 读取 `claude-progress.md`。
4. 读取 `feature_list.json`。
5. `git log --oneline -5` 查看最近提交。
6. 运行 `./init.sh`。
7. 检查基础 smoke test 或端到端路径是否已经损坏。

然后**只选择**一个未完成功能，围绕它工作，直到它被验证通过，或被明确记录为 blocked。

## 规则

- 同一时间只能有一个 active feature（`feature_list.json` 中只有一个 `in_progress`）。
- 没有可运行证据时，不要声称完成。
- 不要通过重写功能清单来隐藏未完成工作。
- 不要为了"看起来完成"而删除或削弱测试。
- 以仓库内文件作为唯一事实来源。

## 必需文件

- `feature_list.json` — 功能状态追踪器
- `claude-progress.md` — 会话进度日志
- `init.sh` — 标准启动与验证路径
- `session-handoff.md` — 需要简短交接时使用

## 完成门槛

只有在要求的验证成功且结果被记录后，功能状态才可以切换到 `passing`。

## 结束前

1. 更新进度日志。
2. 更新功能状态。
3. 记录仍然损坏或未验证的内容。
4. 在仓库可安全恢复后提交。
5. 给下一轮会话留下干净的重启路径。
