# 评审评分表

> 会话结束后或到达里程碑时，用此表评估 agent 的工作成果是否合格。
> 每个维度 0-2 分。

## 评分维度

### 1. 正确性 — 实现出来的行为是否符合目标功能

- **0** — 不符合需求，核心逻辑错误
- **1** — 基本符合但有遗漏的边缘情况
- **2** — 完全符合需求，包括边缘情况

### 2. 验证 — 要求的检查是否真的跑过，并留下证据

- **0** — 没有跑验证，或验证命令被注释/跳过
- **1** — 跑了验证但证据不完整（如只跑测试没跑 lint）
- **2** — 所有要求的验证都跑过，证据记录在 `feature_list.json` 或进度日志中

### 3. 范围纪律 — 是否基本保持在选定功能范围内

- **0** — 做了多个功能，或改了无关文件
- **1** — 基本在范围内，但有少量越界
- **2** — 严格只在选定功能范围内

### 4. 可靠性 — 结果是否能在重启或重跑后继续工作

- **0** — 需要手动修复才能跑通
- **1** — 能跑通但有偶发失败
- **2** — `./init.sh` 能一次跑通，无报错

### 5. 可维护性 — 代码和文档是否清楚到足以交给下一轮会话

- **0** — 代码混乱，没有注释，下一轮看不懂
- **1** — 代码可读，但关键决策没有记录
- **2** — 代码清晰，关键决策记录在进度日志或交接文件中

### 6. 交接准备度 — 新会话是否能只靠仓库内文件继续推进

- **0** — 需要问人或翻聊天记录才能知道下一步做什么
- **1** — 进度日志有更新但不够具体
- **2** — `claude-progress.md` + `feature_list.json` 足够让新会话无缝继续
