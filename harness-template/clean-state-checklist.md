# 收尾检查清单

> 每次会话结束前逐项检查，确保仓库处于下一轮可以直接开工的状态。

## 检查项

- [ ] **启动路径可用**: `./init.sh` 能跑通，不报错
- [ ] **验证能跑**: 测试命令 (`npm test`) 通过或已知的失败已记录
- [ ] **进度日志已更新**: `claude-progress.md` 反映了本轮实际进展
- [ ] **功能清单准确**: `feature_list.json` 真实反映了 passing 和未验证的边界（没有假 passing）
- [ ] **没有半成品**: 所有修改要么已提交，要么有明确的记录说明为什么未提交
- [ ] **可恢复**: 下一轮会话不需要人工修复就能继续

## 如果某项未通过

- 停下来修好它，再结束会话
- 如果修不好，在 `claude-progress.md` 中详细记录问题，并确保下一轮能复现

## 常见遗漏

| 遗漏 | 后果 | 修复 |
|------|------|------|
| 进度日志没更新 | 下一轮不知道从哪开始 | 补上 `claude-progress.md` |
| 功能清单没改 | agent 可能重复做同一功能 | 更新 `feature_list.json` 状态 |
| 测试未提交 | 下一轮以为测试是旧的 | `git add` 并提交 |
| 临时文件没清理 | 干扰下一轮 init.sh | 删除临时文件 |
